
这是一个基于 Python 的数据脱敏工具，专为处理包含复杂结构的 JSON 文件设计。它不仅能处理标准字段，还能**深度扫描并脱敏嵌入在长字符串、日志或转义 JSON 中的 IP 地址**。

## 核心特性

* **🛡️ 穿透式 IP 扫描**: 不依赖字段名，全局扫描所有字符串内容，精准识别并替换隐藏在复杂文本（如 `Log`, `Config`, `Url`）中的 IPv4 地址。
* **🚫 IP 格式破坏**: 将 IP 地址脱敏为等长的**十六进制乱码**（非 IP 格式），彻底消除 IP 特征，防止下游系统误读。
* **🔗 跨文件一致性**: 基于 `SHA256(值 + Salt)` 算法。相同的原值在不同文件中永远被脱敏为相同的密文，保留数据的关联分析能力。
* **📏 长度保持**: 脱敏后的字符长度严格等于原字符长度，最小化对下游数据库字段长度限制的影响。
* **🆔 UUID 智能处理**: 识别 UUID 格式，脱敏后保留其 `8-4-4-4-12` 的结构特征。

## 脱敏原则 (Masking Principles)

| 原则 | 说明 | 示例 (原值 -> 脱敏值) |
| --- | --- | --- |
| **嵌入式 IP 优先** | 无论字段名叫什么，内容中的 IP 都会被优先挖除并替换为乱码。 | `"host=192.168.0.1"` -> `"host=a1b2c3d4e5"` |
| **IP 格式破坏** | IP 被替换为十六进制字符，不再保留点分十进制格式。 | `10.0.0.1` -> `a1b2c3d4` |
| **敏感字段覆盖** | 特定字段（如 `Id`, `Image`）在 IP 处理后，会进行全量字符串脱敏。 | `"user_id": "alice_01"` -> `"user_id": "f9e8_d7"` |
| **UUID 保留** | 若字段值为 UUID 格式，脱敏后保留分隔符。 | `550e...` -> `1a2b...` |
| **统计豁免** | 统计类字段保持原样，不脱敏。 | `totalNum`, `pageSize` 保持不变 |
